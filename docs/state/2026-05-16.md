## Scope decisions noted (M8-04, M8-05, M8-09 — not ADRs)

Three scope decisions surfaced during M8 that are noted here for clarity but were not promoted to ADRs because they are about scope/sequencing rather than runtime architecture:

- **M8-04 — Skill server-side expansion runs BEFORE @file expansion.** The route's POST handler runs `expandSkillPrompt` first; the resulting text feeds into `runTurnInBackground` where the existing T3 `@file:` expansion runs. This composes naturally: a skill template containing `@file:foo.md` gets the file inlined like a hand-typed prompt. Why not an ADR: this is the natural sequence (skill is template-time, @file is content-time); reversing it would require the T3 hop to also re-template (added complexity, no benefit).
- **M8-05 — `/expand [N]` is TUI-local; no server route added.** The TUI's ring buffer holds the last N tool blocks; `/expand [N]` re-renders block N with no truncation. No server round-trip needed because the TUI already has the tool result content in memory. Why not an ADR: this is a UI implementation choice (TUI owns its render state); a server route would duplicate the ring-buffer state on the server with no rendering benefit.
- **M8-09 — Real-Anthropic smoke deferred from T8 to a separate session.** M8's autonomous backend tests use the mock provider only; the cost-bounded real-Anthropic smoke pass that M7 ran (`scripts/m7-real-smoke.ts`, ~$0.000822 cost) was not folded into T8. Why not an ADR: it's a session-sequencing choice (real-API smoke is its own session per `docs/conventions/testing-log.md` + the M7 precedent), not a runtime architecture choice. The plan's "Post-M8 backlog audit" section flags it as a follow-up; estimated $0.005 budget.

## Behavioral notes worth knowing next session

1. **Router-mode is now functional in server mode.** Set `userSettings.router.{localProvider, frontierProvider}` (and optionally `defaultLane`, `localModel`, `frontierModel`), then run with `--provider router`. The runtime wraps both providers in a single RouterProvider that dispatches per call based on the provider's internal classifier. Audit logs land at `<harnessHome>/router/audit.jsonl`.

2. **Capture and replay are turn-stable.** A fixture captures provider events + tool results; replay re-emits exactly the same events. The fixture's `meta.sessionId` is `'pending'` because capture happens at runtime boot (before any session is minted) — replay creates fresh session ids. If you need session-stable fixtures (e.g., for resume semantics), capture must be invoked per-session via a future M9 surface.

3. **`@file:` expansion runs in the ROUTE, not in the orchestrator.** The expanded text is what lands in `sessionDb.messages` AND what the model sees. A resumed session loads the already-expanded text; the orchestrator never sees a `@file:` token. If you need late-binding @file references (re-expand at query time), that's a separate surface (deliberately not provided in M8).

4. **Skills load once at boot.** Adding a new skill mid-session requires a `sov` restart (or a future `/skills reload` slash command — not in M8). The byName lookup is UNFILTERED for dispatch but the per-turn `ctx.skills` view is FILTERED — the model's view of "available skills" matches the active toolset.

5. **The TUI's `/skillname` interception is local.** If the TUI's cache is stale (e.g., a skill was added on disk after the cache loaded), the interception will miss and the slash will be sent as plain text. The server's `kind: 'skill'` dispatch protects against this by validating the skill name; missing names return 400.

6. **`stall_detected` rides the trace recorder, not the StreamEvent union.** The trace event is the source of truth (a single `query()` emission); the SSE wire event is a projection added by the route's `traceRecorder` decoration. Other trace events stay file-only — `stall_detected` is the only one with a wire counterpart today.

7. **`session_summary` extension fields are OPTIONAL.** The schema marks `tokens`, `toolCalls`, `toolOk`, `toolErr`, and duration fields optional. M7-vintage consumers parse the event unchanged; M9-vintage consumers can use the new fields. A best-effort `getSessionMetrics` read failure still emits the M7-shape event.
