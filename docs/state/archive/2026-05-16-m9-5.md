## Behavioral notes worth knowing next session

1. **TOML loader doesn't validate hex codes.** lipgloss accepts the color verbatim. A user file with `primary = "not-a-color"` will render whatever lipgloss does with that string. Validation candidate for M9.6.
2. **Built-ins always win** (ADR M9.5-01). Saving `~/.harness/themes/dark.toml` does NOT override the built-in dark. To customize, save under a different name.
3. **The `name` field in TOML is mandatory.** Empty / missing name returns an error from `LoadFromFile`. Filename is a hint but the theme's self-declared identity wins for `Theme.Name`.
4. **Config writes are atomic.** Temp-file-then-rename pattern. Partial failure leaves the original config valid.
5. **Boot errors are non-fatal.** Missing config, unreadable file, malformed JSON, unknown theme name all silently fall back to `dark`. Unknown theme name surfaces a dim marker on first WindowSizeMsg; other failure modes are silent (config-not-existing is the common first-run case).
6. **The Sovereign palette uses GitHub Dark base colors** (`#0d1117`, `#161b22`) with a cooler cyan primary (`#58a6ff`) and the same green/yellow/red accents. Restrained, professional, AI-tooling aesthetic.
7. **Tokyo Night Storm** ships matching the upstream community palette exactly. No deviations.
8. **Inherited notes from prior snapshots** (still valid):
   - Per-session `SessionContext` cluster on `Runtime.sessionContexts`; lazy-built; torn down via `disposeSession`.
   - Trajectory writes on disposal, not per-turn.
   - `runtime.dispose()` order — per-session → router audit logger → MCP pool → approval queue → sessionDb.
   - Compaction creates a new session id; the TUI hops `m.sessionID`.
   - On-disk `SessionDb` at `<harnessHome>/sessions.db`. Override with `--db <path>`.
   - Resume requires a valid UUID; unknown id → `SessionNotFoundError`.
