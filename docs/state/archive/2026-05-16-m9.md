## Behavioral notes worth knowing next session

1. **Theme constructor injection is value-typed.** A component's `theme.Theme` field is a copy; mutating it doesn't affect other components. Theme changes propagate via explicit `SetTheme(t)` calls on the consumer (`m.transcript.SetTheme(m.theme)`).
2. **Renderer fallbacks never propagate errors.** Markdown parse error → Plain. Chroma error → Plain. Diff parse failure → Plain on the raw text. The TUI never crashes on model-produced garbage.
3. **DiffView focus is orthogonal to /expand ring buffer (ADR M9-07).** The two never interact. `j`/`k` work ONLY inside a focused diff view; the rest of the time the keys fall through to the prompt.
4. **`Esc` is overloaded** — at the outermost level it quits the TUI; when permission modal is open, it denies; when autocomplete popup is visible, it dismisses; when DiffView is focused, it defocuses. The order in `Update` resolves the overload top-down: modal → diff-focus → autocomplete → quit.
5. **`status_update` events fire 2 (or 3) times per turn.** Start (`streaming: true`), end (`streaming: false` with tokens + cost), and on error (`streaming: false` without tokens). M9.5 may add mid-turn updates for slower-streaming providers if visible jitter becomes a problem.
6. **The goodbye card replaces the entire frame.** When `m.goodbyeSummary` is set (after `session_summary` event arrives), the View() returns just the card — no transcript/prompt/statusline below it. The TUI is about to exit anyway; the card is the last visible state.
7. **`/theme` is dispatched purely client-side (ADR M9-12).** No server round-trip; no persistence in M9. The user sees the change immediately; restarting the TUI loses the choice.
8. **Inherited notes from prior snapshots** (still valid):
   - Per-session `SessionContext` cluster on `Runtime.sessionContexts`; lazy-built via `getSessionContext`; torn down via `disposeSession`.
   - Trajectory writes on disposal, not per-turn.
   - `runtime.dispose()` order — per-session → router audit logger → MCP pool → approval queue → sessionDb.
   - Compaction creates a new session id; the TUI hops `m.sessionID` (ADR M6-01).
   - On-disk `SessionDb` at `<harnessHome>/sessions.db`. Override with `--db <path>`.
   - Resume requires a valid UUID; unknown id → `SessionNotFoundError`.
   - Preflight runs once per `--ui tui` boot. `--no-preflight` skips it.
   - Ink TUI preserved on `origin/archive/ink-tui-2026-05-12`. Phase 16.0a daemon code remains dormant in `src/daemon/` and `src/channels/`.
