## `sov dispatch` — what it is + when to use it

A first-principles headless slash-command surface for mechanical regression testing of slash dispatch logic. Reads commands from stdin (one per line), prints output framed by `--- ready ---` (boot complete) and `--- end-of-turn ---` (per-command separator), exits on EOF or `/quit`.

**Works:** read-only commands — `/help`, `/about`, `/cost` (zero-state), `/model`, `/config`, `/permissions`, `/tools`, `/skills`, `/context-budget`, `/expand`, `/theme`, `/security-audit`, `/summarize`, etc. Plus state-shaped commands that gracefully no-op in dispatch context — `/clear` returns "history cleared (dispatch mode — no in-memory history)".

**Errors informatively:** `/compact`, `/rollback`, `/resume`, `/tasks`, `/review`, `/stats`, `/export`, `/copy`, `/init` (prompt command), `/commit` (prompt command). Message: "dispatch mode does not maintain a session DB — /X requires the interactive REPL".

Use case: fast deterministic regression testing (`echo "/help" | sov dispatch` ≈ 1s, $0) when you want to verify the dispatch pipeline without spinning up the agent loop.
