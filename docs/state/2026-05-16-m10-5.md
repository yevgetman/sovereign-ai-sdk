## Where we are

**Phase 16.1 M10.5 is shipped — backlog #40 closed; M11 unblocked.** The HIGH gap surfaced in the M10 audit (slice 1) — that the legacy terminalRepl slash-command stack (`COMMANDS`, `buildCommandRegistry`, `dispatchSlashCommand` from `src/commands/registry.ts`) had no equivalent in `--ui tui` — is now closed by:

- **One new server route:** `POST /sessions/:id/commands { name, args }` at `src/server/routes/commands.ts`. Builds a per-request `CommandContext` via the new factory in `src/server/commandContext.ts` (mirrors `src/cli/dispatchCommand.ts:46+` server-flavored). Returns a JSON envelope `{ output, error?, sideEffects? }`. Closure-based side-effects collector lets the route surface mutations (modelChanged, exitRequested) back to the TUI.
- **One new Go transport client:** `packages/tui/internal/transport/commands.go` mirrors the wire shape.
- **One new Go slash router:** `packages/tui/internal/app/slashrouter.go` + ENTER-handler integration in `app.go`. Leading-slash inputs that don't match `/theme` (client-side), `/compact` (existing route), `/skills <verb>` (existing route), `/expand`, or a known skill name now route to `POST /commands`. The model never sees them as plain text again.

Approach A was selected: single unified `/commands` route; existing dedicated routes preserved with their richer result shapes (M9.6's `/skills` cache-invalidation contract intact). ADRs M10.5-01..03 cover the architectural choices.

## Behavioral notes worth knowing next session

1. **Slash router precedence in the TUI** (top-down in `app.go` ENTER handler): `/theme` (client-side) → `/compact` (dedicated route) → `/expand` (client-side ring) → `/skills <verb>` (dedicated) → known skill name (`/turns kind:'skill'`) → generic `/commands` route → fallthrough (normal turn POST). The fallthrough still applies for non-slash text and for slash inputs the new dispatcher doesn't recognize (e.g., a typo's error envelope is rendered in warning style).
2. **CommandContext is per-request** in server mode. Each `POST /commands` call constructs a fresh CommandContext from the live Runtime + SessionContext. This is intentional — settings cascade (permissions, hooks) re-reads on each command so `sov config set <key> <val>` mid-process takes effect on the next command without restart. Matches REPL semantics.
3. **`runtime.model` mutates** on `/model <name>`. The mutation persists for subsequent turns AND for subsequent `/cost`/`/stats` reads. The Go TUI's StatusLine doesn't dynamically render model (M2 fixed-field design); the change is visible only via the command's transcript output. Future M11+ polish could thread a SetModel into StatusLine.
4. **/clear, /rollback informative outputs reference backlog item #41 explicitly** so users see the M10.5 scope-out and tracking. Pre-`output` style (not `error`) — they're informational, not failures.
5. **Prompt-type commands** (e.g., `/commit`) return their generated prompt text wrapped in a "paste as a message to execute" preamble. Server-mode v1 doesn't auto-route them to /turns — copy/paste is the v1 UX. M11+ could add `sideEffects.promptToSend` for auto-routing.
6. **Inherited notes from prior snapshots** (still valid):
   - Per-session `SessionContext` cluster; lazy-built; torn down via `disposeSession`.
   - Trajectory writes on disposal, not per-turn.
   - `runtime.dispose()` order — per-session → router audit logger → MCP pool → approval queue → sessionDb.
   - Compaction creates a new session id; the TUI hops `m.sessionID`.
   - On-disk `SessionDb` at `<harnessHome>/sessions.db`. Override with `--db <path>`.
   - Resume requires a valid UUID; unknown id → `SessionNotFoundError`.
   - Preflight runs once per `--ui tui` boot.
   - Constructor-injected themes (no globals); 4 built-in palettes (dark, light, tokyo-night, sovereign).
