### Workflow

- **`.github/workflows/release.yml`** (new, 189 lines) — four-job workflow. Preflight passes the version through as an output for downstream jobs. `actions/upload-artifact@v4` + `actions/download-artifact@v4` (`pattern: tarballs-*`, `merge-multiple: true`) carry tarballs between jobs. `concurrency: { group: release-${version}, cancel-in-progress: false }` blocks duplicate-trigger races. The `release` job's upload step exports `GH_TOKEN: ${{ secrets.SOV_RELEASES_TOKEN }}` only for the upload command, scoping the secret as tightly as possible. Native `--version` smoke runs on the matching native target per build job (darwin-arm64 on macos-14, linux-x64 on ubuntu).

## Behavioral notes worth knowing next session

1. **Tag push is the release trigger.** Pushing a `vX.Y.Z` tag on the private repo fires the workflow. There's no separate "publish" button. To do a release: bump package.json, commit, push, tag, push tag.
2. **CHANGELOG.md lives in sov-releases**, not in `sovereign-ai-sdk`. The workflow's `release` job checks out sov-releases for `--notes-file ${SOV_RELEASES_PATH}/CHANGELOG.md`. Update the CHANGELOG entry **before** tagging, push to sov-releases, then tag-push the private repo.
3. **`SOV_RELEASES_TOKEN` is a fine-grained PAT** scoped to sov-releases only. It expires in 1 year (max for fine-grained PATs). Regenerate before expiry; the secret name + scope stay the same.
4. **`scripts/release.ts` still works.** When CI is broken, the local cut path is intact. The script's tag-and-push at the end will fire CI for the same tag, but the upload step is idempotent — CI sees the already-published release and exits 0.
5. **`package.json` version must match the pushed tag.** Preflight aborts with a clear error if `package.json` says `0.6.0` but the tag is `v0.7.0` (or vice versa). This catches the common foot-gun of forgetting to bump before tagging.
6. **macOS runners are 10× billable** on private repos. Each release uses ~30–50 billable macOS-minutes. Acceptable at the author's release cadence; the alternative (cross-compile-from-Linux-to-darwin-arm64) is unvalidated and was deferred.
7. **`workflow_dispatch` is the recovery path.** If a tag-push triggered workflow fails, fix the issue + use `gh workflow run release.yml -f version=vX.Y.Z -f dry-run=true` to re-run without re-tagging. Dry-run for safe iteration.
8. **GitHub Actions billing matters.** The first dispatch on this machine failed at runner-startup because of an account-level payment issue. Worth monitoring billing periodically; macOS runners burn minutes fast.
