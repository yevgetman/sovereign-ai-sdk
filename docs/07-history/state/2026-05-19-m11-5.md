## Behavioral notes worth knowing next session

1. **Picker commands branch on `ctx.requestPicker`.** ADR M11.5-01. Server-mode contexts populate it via `buildServerCommandContext` at `src/server/commandContext.ts:178-188`; REPL/CLI contexts leave it undefined. The migration pattern in `runModelPicker`, `runResumePicker`, `runExport` is identical — check capability before falling back to `pick()`.
2. **Picker resolution is stateless on the server.** ADR M11.5-03. Selecting a row in the TUI dispatches `/<command> <value>` as a fresh M10.5 call. No server-side continuation state; no `pickerId`; no timeouts to manage. If the user opens two TUIs against the same session, each picker is independent.
3. **The TUI locks input while the picker is open.** `packages/tui/internal/app/app.go` routes ↑/↓/Enter/Esc to the picker; all other keys are absorbed (mirrors the permission-modal lock pattern). Esc emits a quiet dim `(cancelled)` marker; Enter dispatches the value with a running indicator that's popped by the second response.
4. **Empty `items` payloads are defensive.** Both `PickerCard.Selected()` and the app's Enter handler check `ok`. The server's `/model` early-returns "no preset models" before emitting the side-effect, so an empty payload shouldn't reach the TUI in practice — but the defense is there.
5. **Behavior change in legacy non-TTY callers.** With the migration, the empty-state check (`sessions.length === 0`, `messages.length === 0`) now runs BEFORE the TTY check on `/resume` and `/export`. Empty-state callers see "no recorded sessions" / "nothing to export" instead of "requires a TTY" — more actionable diagnostic. `/model` behaves the same: missing-provider check before TTY.
6. **Spacing fix (T8) lives in `app.go:View()` only.** Bumped the gap from `out += "\n" + prompt` to `out += "\n\n" + prompt`. One-line change addressing `~/Desktop/ux2.png` feedback.
