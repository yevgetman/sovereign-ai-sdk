## Where we are

**Phase 16.1 M11 is shipped — bare `sov` now boots the Go Bubble Tea TUI by default.** The default-flip has landed safely behind four user-facing escape hatches plus a soft-degradation safety net:

- **CLI flag** — `sov --ui repl` (highest precedence)
- **Env var** — `SOV_UI=repl sov` (mid precedence)
- **Config field** — `sov config set ui.surface repl` then bare `sov` (lowest precedence; persistent across shells)
- **Auto-fallback** — if `sov-tui` binary is missing, bare `sov` falls back to the readline REPL with a one-line stderr warning pointing at the remediation (`bun pm -g trust @yevgetman/sov && sov upgrade`).

The flip closes the milestone arc that began with M10's parity audit and M10.5's slash-command dispatcher route. Per ADR M10-03's severity-classified disposition rule, no CRITICAL/HIGH/MEDIUM gaps remained — M11's audit also returned PASS with a single LOW doc-polish finding (fixed inline).

## Behavioral notes worth knowing next session

1. **Default surface is now TUI.** Bare `sov` (or `sov chat`) launches `runTuiLauncher` via `src/main.ts:232`. Users who never passed `--ui tui` before will see the new surface on their next session unless they set a persistent opt-out. The deprecation-notice for `sov chat` still fires (unrelated to M11).
2. **Surface resolver lives at `src/cli/surfaceResolver.ts`** — a pure 76-line module. The caller (currently only `src/main.ts:199-256`'s default-action handler) passes the CLI flag, env, and config explicitly; no I/O happens in the resolver. Reusable for any future entry point that needs the same precedence semantics.
3. **Missing-binary fallback fires upstream of `runTuiLauncher`.** `src/main.ts:221-230` calls `findTuiBinary()` and downgrades `effectiveSurface` to `'repl'` before invoking either branch. The internal `tuiLauncher.ts:124-136` null-binary guard remains as a defensive belt-and-suspenders check for direct importers (tests, future callers); it's unreachable from the bare-`sov` flow post-M11.
4. **Resolver source-tracking.** `resolveSurface()` returns `{ surface, source }` where `source ∈ {'cli', 'env', 'config', 'default'}`. The `source` field isn't user-visible today but is captured for future debug logging or `/status` output. Tests pin the source for each precedence path.
5. **Postmortem Rule 4 still honored at the strongest level.** Three explicit opt-outs (CLI, env, config) + auto-fallback. No user is forced into TUI with no out. If a regression surfaces post-M11, `sov config set ui.surface repl` is a one-time, immediate, persistent workaround.
6. **Inherited notes from prior snapshots** (still valid):
   - Per-session `SessionContext` cluster; lazy-built; torn down via `disposeSession`.
   - Trajectory writes on disposal, not per-turn.
   - `runtime.dispose()` order — per-session → router audit logger → MCP pool → approval queue → sessionDb.
   - Compaction creates a new session id; the TUI hops `m.sessionID`.
   - On-disk `SessionDb` at `<harnessHome>/sessions.db`. Override with `--db <path>`.
   - Resume requires a valid UUID; unknown id → `SessionNotFoundError`.
   - Preflight runs once per `--ui tui` boot.
   - Constructor-injected themes (no globals); 4 built-in palettes (dark, light, tokyo-night, sovereign).
