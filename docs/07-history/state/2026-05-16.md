## Behavioral notes worth knowing next session

1. **Mouse click region detection** uses `vp.YOffset` from bubbles viewport to translate viewport-relative Y → absolute Y in `transcript.lines`. Scrolled transcripts work correctly because the offset is included.
2. **Stall badge generation counter** ensures `tea.Tick(5s)` from an earlier stall can't accidentally clear a refreshed badge. The closure captures the gen at schedule time; on fire, it compares against `m.stallGeneration` — mismatch means "newer stall extended the badge; ignore this tick."
3. **`/skills` subcommand parser runs BEFORE `matchSkillSlash`**, so a user could never define a skill named "skills" that would shadow the subcommand. This is the intentional precedence per ADR M9.6-03.
4. **`compaction_complete` triggers `fetchSkillsCmd`** automatically. The cmd captures `m.sessionID` at construction time (which is the post-pivot child id by then), so the refetch targets the new session correctly.
5. **Hex validation is per-field, not whole-file.** A user TOML with 12 valid colors + 1 bad ("red") produces a 12-color-valid theme with `Dark()` filling the bad field. No error surfaced to the user (potential M10+ enhancement).
6. **`--no-mouse` is a `sov-tui` flag**, not a `sov` flag. The TS-side launcher would need to thread it through if `sov --ui tui --no-mouse` should work; currently `sov-tui --no-mouse` is the only path.
7. **Inherited notes from prior snapshots** (still valid):
   - Per-session `SessionContext` cluster; lazy-built; torn down via `disposeSession`.
   - Trajectory writes on disposal, not per-turn.
   - `runtime.dispose()` order — per-session → router audit logger → MCP pool → approval queue → sessionDb.
   - Compaction creates a new session id; the TUI hops `m.sessionID`.
   - On-disk `SessionDb` at `<harnessHome>/sessions.db`. Override with `--db <path>`.
   - Resume requires a valid UUID; unknown id → `SessionNotFoundError`.
   - Preflight runs once per `--ui tui` boot.
   - Ink TUI preserved on `origin/archive/ink-tui-2026-05-12`. Phase 16.0a daemon code remains dormant.
