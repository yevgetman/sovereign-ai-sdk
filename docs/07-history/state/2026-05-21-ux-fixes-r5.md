## Where we are

Three rounds of UX feedback shipped 2026-05-21:

### Round 3 (commit `e2c8b63`)
- **Textbox auto-grow**: replaced `bubbles/textinput` (single-line, scrolls horizontally) with `bubbles/textarea` (multi-line, auto-grow up to 8 rows). Alt+Enter / Ctrl+J insert real newlines; Enter still submits. `Prompt.Height()` exposes the current row count so `app.go`'s `recomputeLayout` resizes neighboring chrome dynamically.
- **Copy/paste**: mouse capture flipped OFF by default. Old `--mouse` opt-in flag, `--no-mouse` deprecated.
- **Splash polish**: border padding `Padding(0,1) → Padding(1,2)`. Title `Bold(true)` only (no Foreground, inherits terminal default per `tui-color-rendering.md`). Model + provider plumbed end-to-end via new `--model`/`--provider` CLI flags from `tuiLauncher.ts`. Statusline placeholders changed `"?"` → `""`.
- **Spinner alignment**: bottom-weighted Braille rotation (`⢀⣀⡀⡄⠄⠤⠠⢠`) so the glyph sits at the text baseline of "Thinking…" instead of overhanging.
- **Markdown wrap**: `List.Indent = 2` so bullet continuation lines hang under the bullet text. Table cells skip the backtick-wrap pass so inline-code ANSI doesn't leak across cell boundaries.

### Round 4 (commit `2976466`)
- **Paste abstraction**: `Prompt.RegisterPaste(content)` replaces ≥ 2-line or ≥ 200-char pastes with `[Pasted text #N +M lines]`. `Prompt.ExpandPastes(value)` reconstitutes the real content on Enter so the server sees full text. Below-threshold pastes inline-insert.
- **ESC turn-cancel**: new server-side `ServerEventBus.setCurrentTurnAbort` / `cancelCurrentTurn` methods, new `POST /sessions/:id/cancel` route, runtime composes `AbortSignal.any([bus, turn])` so per-turn cancellation doesn't tear down the bus. TUI's ESC during streaming POSTs the route, shows `(interrupted by user)` immediately, and suppresses the consequent `turn_error` once via `m.userCancelledTurn`. Ctrl+C still tears down the session.
- **Prompt prefix first-line-only**: `SetPromptFunc(2, lineIdx => idx==0 ? "› " : "  ")` so the bullet appears once and wrapped continuation rows indent two spaces.
- **Scroll keys** (later dropped in round 5): added `pgup/pgdown` and `shift+up/shift+down` to scroll the transcript viewport. Round 5 deleted these in favor of native terminal scroll.

### Round 5 (commit `bf47564`) — the architectural pivot
- **Drop `tea.WithAltScreen()`**: the terminal now owns scrollback. Wheel + trackpad scroll and click-drag text selection work natively without any in-TUI coordination. This is the same model Claude Code uses.
- **Drop mouse capture entirely**: `--mouse` / `--no-mouse` demoted to no-op back-compat shims.
- **New `LiveRegion` component** (`packages/tui/internal/components/liveregion.go`): bottom-of-screen, non-scrolling. Holds the in-flight streaming assistant card (`AppendAssistantDelta` / `EndAssistantCard`), the spinner frame (`SetSpinner` / `ClearSpinner`), and a "…running /name" indicator (`SetRunningCommand`).
- **`tea.Println` for committed history**: all permanent content (user messages, assistant cards, tool results, system messages, splash, boot notices, slash-command output, compaction markers, turn errors) emits via `m.print` / `m.printUser` → drained as a single `tea.Println` Cmd via `m.respond`. Lines land in the terminal's native scrollback above the live View.
- **Tool cards print fully expanded**: click-to-expand interaction retired (scrollback is immutable). `/expand N` still re-renders the raw payload from the local ring buffer.
- **Round-4 PgUp/PgDn/Shift+arrow scroll bindings deleted**: terminal owns scroll now.
- **User-message wrap + truncate**: `printUser` wraps via `lipgloss.NewStyle().Width(w).Render` at the terminal width minus marker column; continuation rows indent two spaces. Above `userMessageDisplayCap = 1500` chars, the visual echo truncates with a dim italic `…[+N chars]` marker. The full text still ships in the actual turn.
- **Paste flush immediately**: round 4 misread bubbletea's bracketed-paste API and accumulated across multiple KeyMsgs, waiting for a non-paste event to flush — that's why pastes only appeared after typing or hitting space. Bubbletea actually returns ONE KeyMsg per paste with `Paste=true` and `Runes` holding the entire pasted content. Round 5 flushes that single KeyMsg immediately in the same Update tick.
- **`transcript.go` retained as dead production code**: still referenced from `recomputeLayout`'s legacy `SetSize` call and the `New()` constructor; its viewport / `lines[]` / `toolCards` map are unused. Cleanup deferred to a follow-up pass to keep the round-5 diff focused.

## Behavioral notes worth knowing next session

1. **Wheel scroll, trackpad scroll, click-drag selection ALL work natively now.** No TUI logic; the terminal handles it. This matches Claude Code, opencode, and every other modern terminal harness.
2. **Session history lives in terminal scrollback.** Once printed via `tea.Println`, content is owned by the terminal — can't be re-themed, edited, or removed from inside the TUI. `/theme <name>` mid-session only affects the in-flight live region; prior scrollback retains its original styling. This is the same constraint every chat-style TUI has.
3. **Tool cards are static once printed.** No click-to-expand, no collapse. Use `/expand N` to re-render the Nth-most-recent tool's raw payload below the prompt.
4. **`m.print` / `m.printUser` / `m.respond` is the only path to scrollback.** New code that needs to surface content to the user uses `m.print(rendered)` (queued) and returns through `m.respond(cmd)` (drained). Direct `tea.Println` calls would also work but bypass the queue's order guarantee.
5. **`m.live.SetSpinner` / `ClearSpinner` / `SetRunningCommand` / `AppendAssistantDelta` / `EndAssistantCard` are the live-region API.** `EndAssistantCard` returns the rendered card string; callers `m.print(rendered)` to commit it to scrollback.
6. **Paste arrives as ONE KeyMsg with `Paste=true`.** Don't try to accumulate across events — round 4 made this mistake. The Runes field holds the entire pasted content including newlines.
7. **`--mouse` and `--no-mouse` are no-op shims.** Kept for back-compat with older launcher invocations; mouse capture is gone for good in round 5.
